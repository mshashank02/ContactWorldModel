# ShadowHand-TQC

Touch-sensor-based in-hand manipulation with Shadow Hand, trained using **TQC + HER**.

## Current Status

- Supports standalone environment generation via `pipeline_generate.py`.
- Supports custom mesh tasks via `--task /path/to/object.msh`.
- Supports deformable custom-object generation via `--deformable` (when `--task` is a `.msh`).
- Training entrypoint is `ShadowHand_TQC.py` with dynamic XML loading (`--xml-path`).

## Compatibility (validated)

Use this package pair for custom deformable XMLs generated by this repo:

- `mujoco==3.3.1`
- `gymnasium==1.2.0`
- `gymnasium-robotics==1.4.2`

The repo also includes a dynamic env compatibility fix so training works with stable XML timestep `0.0001`.

## Install

```bash
conda create -n ShadowHand python=3.11 -y
conda activate ShadowHand
bash install.sh
```

## Quick Training (existing generated XML)

```bash
WANDB_MODE=disabled python ShadowHand_TQC.py \
  --env-id custom_train_run \
  --xml-path /home/mshashank02/ShadowHand-TQC/generated/custom_sphere_macro_high_rough_high_obj_100_0.5_1/manipulate_custom_sphere_macro_high_rough_high_obj_touch_sensors_100_0.5_1.xml
```

## Generate Standalone Env Only

```bash
python pipeline_generate.py --standalone \
  --base assets/hand_base.xml \
  --task block \
  --Ntotal 16 --Rppx 0.4 --Rpt 0.3 \
  --out-root generated --force
```

## Generate Custom Deformable Object Env

```bash
python pipeline_generate.py --standalone \
  --base assets/hand_base.xml \
  --task /absolute/path/to/object.msh \
  --deformable \
  --Ntotal 100 --Rppx 0.5 --Rpt 1 \
  --out-root generated --force
```

Notes:
- `--deformable` is valid only when `--task` points to a `.msh` file.
- Generated deformable settings are aligned to the stable reference config used in this repo.

## Generate + Train in One Step

`generate_and_train.py` builds a standalone candidate and forwards args after `--` to `ShadowHand_TQC.py`.

```bash
python generate_and_train.py \
  --base assets/hand_base.xml \
  --task /absolute/path/to/object.msh \
  --Ntotal 100 --Rppx 0.5 --Rpt 1 \
  --out-root generated --force -- \
  --num-envs 1 --seed 0 --learning-starts 1000
```

## Common Runtime Note: Contact Limit

If you see:

`Too many contacts ... (ncon = 100)`

increase `nconmax` (and optionally `nstack`) in:

- `generated/<candidate>/shared.xml`
- or `assets/shared.xml` (to affect future generations)

Example:

```xml
<size njmax="500" nconmax="1000" ... nstack="1200000" />
```
